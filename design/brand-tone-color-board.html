컬러 시스템 — 가드레일 주도 도출

각 결정을 아티클 기반 가드레일과 측정 가능한 목표로 먼저 고정하고, 색 값은 규칙과 계산이 결정한다. AI의 취향 판단이 개입할 수 있는 지점은 아래 프로토콜로 차단했다.

0. 판단 배제 프로토콜

색이 “선택”되는 3가지 경로를 모두 규칙으로 대체한다.

앵커 색 — 발명 금지

기준 유채색·잉크·배경은 새로 만들지 않는다. 출처는 이미 확정된 내부 브랜드 문서(persona-brand-tone.md)의 값만. 내부에 없는 색(success)은 검증된 오픈소스 스케일(Radix)에서 용도 일치 단계를 차용.

파생 색 — 수식으로만

hover·pressed·disabled·보더·배지·보조텍스트는 전부 “앵커 × 고정 알파”로 도출(M3 상태 레이어 수치, Radix 알파 변형 방식). 중간 톤 hex를 손으로 고르지 않는다.

채택/탈락 — 계산이 판정

모든 텍스트·UI 조합은 WCAG 대비 계산 스크립트로 검증. 4.5:1(텍스트)/3:1(UI) 미달이면 취향과 무관하게 탈락·교체. 아래 표의 PASS/FAIL은 전부 계산 결과다.

1. 가드레일 — 증거 등급제로 재선정

이전 버전은 국제 표준·벤더 스펙·개인 의견·트렌드 논평을 전부 “하드 제약”으로 동급 처리한 방법론 오류가 있었다. 출처의 성격에 따라 등급을 나누고, 등급이 구속력을 결정한다.

등급구속력항목과 출처
전략 축 인간 결정 — 언제든 변경 가능, 변경 시 하위 절차 재적용 브랜드 형용사(차분·정확·투명·정갈) · 앵커 hue 207° · 배경 화이트(확정) · WCAG AA는 목표로만 채택(확정) — 출처: persona-brand-tone.md + 이 대화의 결정 로그
Tier 1
규범 표준		 목표로 채택(전략 축 결정에 따름) — 위반은 차단이 아니라 측정·보고 WCAG 1.4.3/1.4.11 — 단 실제 조항 범위 그대로: 본문 4.5:1 / 대형·볼드 텍스트 3:1 / UI 경계 3:1 / 비활성·부수 텍스트 예외. 이전 버전의 “모든 텍스트 4.5” 일괄 적용은 임의 강화였음
Tier 2
채택 스펙		 채택 선언 후 내부 일관성만 강제 — 채택 자체는 결정사항, 타 시스템과 혼용 금지 M3 상태 레이어(hover 8 · pressed 10~12 · disabled 38) · Radix 알파 변형/스텝 의미론 · “서페이스 solid, 상태 alpha” · 요소 opacity 금지 · 서페이스 문맥 문서화(Improta)
Tier 3
실증 연구		 방향 제시 — 반증 가능, 상반 결과 병기 NN/g 톤 4축·신뢰-추천 상관 · 채도-감정 지각 연구. 주의: 채도↑=제품 성능 지각↑ 연구(ScienceDirect)도 존재 — muted 일변도의 근거는 단정적이지 않음
Tier 4
실무 휴리스틱		 기본값 — 이유를 기록하면 이탈 허용 뉴트럴 1패밀리 · 프라이머리 1~2 · 시맨틱 3~4(Refactoring UI) · 60-30-10 배분 · 계정 배지=뉴트럴
Tier 5
트렌드 논평		 참고자료 — 규칙 아님 AI 퍼플 문제(인디고 240–280° 회피) · warm minimalism · Calm Technology 에세이. 방향 감각으로만 사용

2. 목표 대시보드 — AA는 목표, 차단이 아니라 측정

전략 축 결정(“AA는 목표로만”)에 따라 대비는 통과/미달을 측정해 보고한다. 미달 = 알고 쓰는 선택이 가능하되, 기록이 남는다.

5. 적용 확인 — 셀프 감사 반영

1차 목업을 합성 대비로 재감사한 결과 3건의 자체 위반이 발견되어 수정했다. 위반과 수정도 전부 계산 기준.

감사에서 발견된 위반계산 결과수정
“확인함” 카드 텍스트에 ink@38 사용 — 38%는 비활성 컨트롤 전용인데 읽어야 할 콘텐츠에 적용2.30 FAIL전 텍스트 ink@64로 (4.82 PASS). 낮춤 효과는 보더·굵기·색으로만
일반 버튼 보더에 48% 적용 — 48%는 “경계가 유일한 식별자”인 인풋 전용 규칙규칙 오적용 (1.4.11 예외 대상)버튼 보더 12%로 환원. 48%는 인풋·체크박스에만
“정리됨” 노트: ink7 틴트를 bg 위에 직접 사용내부 ink64 텍스트 4.41 FAILsolid 서페이스 + 12% 보더로 교체 (텍스트 4.82 PASS). ink7 틴트는 white 위에서만(§4)

7. 결정 로그와 열린 결정

확정된 전략 축 결정: 접근성 = WCAG AA를 목표로만 · 블루·그린 버튼안 기각 · 아이덴티티 = 화이트 + 라이트 크림 투피스(진한 크림 금지) · 탁함 보정: 잉크는 중성(#43464A/#26282B)으로 — 웜 잉크(갈색기)가 탁함의 원인이었음 · 크림은 노란기 뺀 #F6F4EF · 순흑·대면적 다크 금지 유지.

  • 프라이머리 투톤 여부 — §5의 두 목업 비교로 결정: 현행 단일톤(#386D98, 5.51) vs 투톤(solid #4585C0 + 텍스트 #386D98). “덜 경직” 방향이면 투톤이 아티클(Radix 구조·명도 진단) 근거가 있는 경로. 더 밝은 경계값(#609ACA, 3.01)도 가능하나 AA 목표선에 걸침.
  • 딱딱함의 비색상 변수 — warm minimalism 문헌이 지목하는 radius·여백·타이포·트랜지션 토큰은 아직 미착수. 색 확정 후 다음 단계.
  • 다크 모드 진행 여부 — 알파 구조상 잉크→화이트 알파 스왑으로 골격은 자동 전환, 시맨틱 3색만 재계산 필요.

8. 브랜드 상징색 선정 — hue 대역 제거법

블루 계열 전체 기각(인간 결정) 후, 상징색은 취향 나열이 아니라 대역 제거로 좁힌다. 기준: ① 시맨틱 3색(success 154° · warning 35° · danger 358°)과의 hue 충돌 ② 경쟁 밀집 대역 이탈(Shortwave·Superhuman = 딥 블루, HEY = 퍼플, Gmail = 4원색 + AI 퍼플 금지대 240–280°) ③ AA 목표 내 텍스트 겸용 명도 존재 ④ 페르소나 형용사(차분·정갈·사적) 부합.

hue 대역판정
빨강 (~0°)제외 — danger(358°)와 충돌, 긴급 연상
주황~노랑 (30~60°)원칙 제외 — warning(35°)과 충돌. 단 “집사의 황동” 은유가 강해 warning 재선정을 감수하는 예외 후보로만 유지
그린 (140~160°)원칙 제외 — success(154°)와 사실상 동일. 단 “완료=브랜드색” 통합안을 감수하는 예외 후보로만 유지
청록 (170~185°)잔존 대역 — 시맨틱과 최소 21° 거리, 경쟁 밀집 대역 밖, Figma·색채심리 문헌에서 “블루의 신뢰 + 그린의 안정, 차분·세련” 계열로 분류
블루 (200~230°)기각됨(인간 결정) + 메일 앱 밀집 대역
인디고~퍼플 (240~280°)제외 — AI 동질화 금지대(Tier 5 참고) + HEY 등 경쟁 사용
후보계산의미 · 대가
A. 스튜어드 틸 #14807A · 177° 텍스트 4.77 · 흰 라벨 4.77 — 단일톤으로 겸용 가능 유일한 무충돌 대역 생존자. 문헌 분류: 신뢰+회복, 차분·세련, 기술·금융·헬스케어에서 신뢰 신호. 대가 없음(시맨틱 유지). success(154°)와 21° — 명도 차로 구분 가능
B. 서재 그린 #256D4F · 155° 텍스트 6.21 · 흰 라벨 6.21 서재·컨시어지·클래식 호텔의 딥 그린 — 집사 은유 부합, 메일 앱 중 부재(차별화 최대급). 대가: success와 동일 hue → “완료·정리됨 = 브랜드색” 통합 재설계 필수
C. 황동 브라스 #6E572E · 38° 텍스트 6.85 · 흰 라벨 6.85 황동 트레이·서재 문구 — 집사 은유 직결, 테크 제품 중 희소. 대가: warning(35°)과 동일 대역 → warning 색 재선정 필수, 웜 메탈이 “업무 도구” 연상과 거리
D. 화이트 + 라이트 크림 투피스 (최종안) #FFFFFF + #F7F3EA · 잉크 #4A4740/#33312C 잉크 on 크림 버튼 11.73 · 소프트 잉크 9.27 · 크림면 경계 1.11 → 보더 필수(22% 알파 적용) 화이트·크림 행 스와치 1·2만 사용, 진한 크림(#EFE6D5) 금지(인간 결정). 유채 액센트 없음 — 아이덴티티는 크림 선택면·라이트 CTA·소프트 잉크의 명도 리듬. 시맨틱 충돌 전무(유채색은 상태 3색뿐). 위험: 무유채 아이덴티티는 브랜드 인지가 형태·타이포에 더 의존
