# 디자인

## 시스템

기본 표면은 제품 UI다. 현재 시각 방향은 Editorial Serif다. 헤딩은 세리프(Newsreader), 한국어 UI 라벨·본문은 Pretendard를 쓴다. 표면은 보더 박스 대신 헤어라인과 좌측 3px 틸 룰(선택 표시 전용)로 구분한다. 아이콘은 24×24 viewBox, stroke 1.5~1.6, `round` cap/join으로 전 화면 통일한다.

구현 기준은 두 곳이다.

- `design/boards/v1/current/shared.css` — 디자인 목업 기준(10화면 공용 스타일시트)
- `development/frontend/src/styles/tokens.css` — 실 구현 기준(위 값을 1:1 포팅)

금지:

- 보라/인디고 계열 AI gradient
- 큰 면적의 채도 높은 배경
- glow 장식
- 컬러 바 중심의 상태 표현(선택 표시용 좌측 3px 틸 룰 제외)
- 마법 같은·말 많은·귀여운·미래적·과잉 확신 인상, AI 마스코트·의인화 카피

## 색상

화이트 기반, 클린 아이보리 보조면, 강한 중성 잉크, 얇은 헤어라인, 단일 틸 포인트를 쓴다.

| 토큰 | 값 | 용도 |
|---|---|---|
| `--bg` / `--surface` / `--panel` | `#FFFFFF` | 배경, 사이드바·상세 패널 등 대면적 표면 |
| `--soft` | `#F6F4EF` | 배지·칩·소형 장식 요소 전용 |
| `--ink` | `#26282B` | 헤딩 |
| `--body` | `#43464A` | 본문 |
| `--label` | `ink 78%` | 구조 라벨(계정명·발신자·섹션/서브 카운트 숫자) |
| `--muted` | `ink 64%` | 순수 부가정보 |
| `--line` / `--line-strong` | `ink 12%` / `ink 22%` | 헤어라인 / 아이보리 면 보더 |
| `--control` | `ink 48%` | 인풋·체크박스 등 보더가 유일한 식별자인 컨트롤 |
| `--selected` | `ink 6%` | 선택 상태 면색(중립 회색, 좌측 틸 룰과 병행) |
| `--hover-ink` | `ink 4%` | hover 레이어 |
| `--teal` | `#14807A` | CTA/Primary 단일톤 |
| `--teal-deep` | `#0E5E59` | 틴트 위 텍스트 스텝(요약 라벨, 배너 info 액션 텍스트) |
| `--teal-soft` | `teal 4%` | 카드 hover/선택 배경 |
| `--success` / `--warning` / `--danger` | `#218358` / `#A16207` / `#CE2C31` | 상태 순간에만 |
| `--on-color` | `#FFFFFF` | 틸/잉크 solid fill 위 텍스트·아이콘 |

`--label`과 `--muted`는 역할이 다르다. `--label`은 화면 구조를 알려주는 텍스트(계정 라벨, 발신자, 카운트 숫자)에 쓰고, `--muted`는 순수 부가정보에만 쓴다.

계정 구분은 아바타로만 한다 — 프로필 사진이 있으면 사진, 없으면 계정 성격별 뉴트럴 아이콘(업무=서류가방, 개인 주 계정=그라디언트 photo 자리, 기타=인물)을 쓴다. 계정 그룹 사이 간격은 여백(18px)만으로 표시하고 룰선·배경 패널을 넣지 않는다.

금지:

- 유채색은 시맨틱 3색(success/warning/danger) + CTA 틸 외 추가 금지
- 계정 배지·계정 아바타에 전용 색상(슬레이트블루 등) 부여 금지 — 아바타 자체가 이미 구분 정보이므로 색 중복 금지
- 아이보리(`--soft`)를 사이드바·상세 패널 등 대면적 배경으로 사용 금지 — 배지·칩·소형 장식 전용
- 시맨틱 틴트(10%) 위에 시맨틱 색 텍스트 금지 — 텍스트는 잉크, 색은 도트/아이콘으로만(배너 info 톤의 틸 액션 텍스트만 기존 요약 라벨 선례로 예외)
- 요소 `opacity`로 비활성·흐림 표현 금지 — 색상 알파 토큰으로 개별 지정(네이티브 체크박스 시각적 숨김 등 접근성 표준 패턴은 예외)
- hover 8% / teal-soft 8% 값 재사용 금지 — 4%로 확정

## 타이포그래피

헤딩과 상세 제목에는 Newsreader 세리프를 쓰고, 한국어 UI 라벨·본문·액션 라벨에는 Pretendard를 쓴다. 상세 원문 본문은 세리프(`--font-serif`), AI 요약 문장은 산세리프(`--font-sans`)로 서체를 대비시켜 원문과 비서 판단을 구분한다.

spacing과 type scale은 4/8/12/16/24/32/48/64px 같은 강제 그리드를 쓰지 않는다. 광학 보정을 포함한 자유 연속형 스케일을 그대로 유지한다. 단 캡션류(발신자, 메타, 카운트 숫자 등 보조 텍스트) 크기는 소수점(11.5/12.5px)이 아니라 정수(11/12px)로 통일한다.

카운트 숫자(사이드바 아코디언, 섹션 헤더)는 세리프 이탤릭이 아니라 산세리프·`tabular-nums`·비이탤릭·`--label` 색으로 쓴다. 원형 배지로 감싸지 않는다.

카드 요약은 "요약: " 같은 접두 문구를 문장에 붙이지 않고, 별도 태그(`sum-tag`)로 분리한 뒤 문장에서는 콜론을 뺀다.

금지:

- 액션 라벨, 메타데이터, 메일 본문에 장식적 display 서체 처리
- 캡션류에 소수점 폰트 사이즈(11.5/12.5px 등)
- 카운트 숫자에 세리프 이탤릭 또는 원형 배지
- spacing/type을 4/8/12/16px류 그리드로 강제 정규화(이 프로젝트의 확정 기준이 아니다)

## 레이아웃

데스크톱 기본 구조는 3-pane 브리핑 레이아웃이다.

- 좌측: 제품 로고, 주요 내비게이션(아코디언), 서비스 계정
- 상단: 메일 계정 스코프, 알림
- 본문: 브리핑 목록
- 우측: 선택한 메일 상세

사이드바 "오늘 브리핑"과 목록 섹션은 네이티브 `<details>/<summary>` 아코디언으로 구성하고 기본 열림 상태를 쓴다. "오늘 브리핑" 최상위 총합 숫자는 표시하지 않고, 하위 항목별 숫자만 보여준다.

첫 화면은 마케팅 페이지가 아니라 실제 앱 화면이어야 한다. 태블릿/모바일에서는 가로 스크롤 없이 구조적으로 접히도록 만든다.

앱 셸(`.app`)은 뷰포트 높이에 고정한다(`height: 100dvh`, `min-height` 아님) — 사이드바/목록/상세 3단이 각각 독립 스크롤되고, 사이드바 하단 서비스 계정 영역이 항상 화면 하단에 고정되게 한다. 사이드바 내비게이션이 넘치면 내비 영역만 자체 스크롤하고 상단 로고·하단 계정 영역은 고정 유지한다.

금지:

- 사이드바 상위 항목에 총합 카운트 숫자 노출
- 목록/사이드바 섹션을 아코디언이 아닌 고정 펼침 구조로 되돌리기
- 모바일/태블릿에서 가로 스크롤 발생

## 컴포넌트 원칙

- 사이드바는 섹션 앵커와 중립 선택 상태를 가진다. "오늘 브리핑" 하위 목록(새 중요 항목 등)의 현재 위치 표시는 배경색이 아니라 글자 굵기(750)로만 한다 — 배경 칩, 세로 연결선 둘 다 없음.
- 상단에는 계정 스코프와 알림 벨만 둔다. 보조 컨트롤에 소프트 배경 칩을 넣지 않는다.
- 메일 카드는 상태 배지, 컬러 바, 카드 내부 액션을 갖지 않는다. 카드는 계정 아바타, 제목, 발신자/시간, 요약(태그+본문)만 담는다. 카드 사이는 구분선이 아니라 12px 여백으로만 나눈다 — 카드 자체에는 테두리를 두지 않고, 선택 상태만 좌측 틸 룰 + 배경 틴트로 표시한다. 계정 아바타는 16px(내부 아이콘 9px), 계정 라벨(11px 캡션)과 시각적으로 맞춘 크기다.
- 상세 패널은 icon-only Gmail 액션(3종: Gmail에서 열기 / Gmail도 읽음 처리 / `…`), 세리프 제목, 요약, Gmail handling 한 줄, 원문 미리보기를 담는다. 헤더와 본문 사이는 헤어라인으로 구분한다.
- icon-only 액션 3종은 24×24 viewBox 안에서 5~19 좌표 구간으로 체급을 통일한다. 외부링크 아이콘은 화살촉이 있는 실제 모양을 쓰고, 케밥(더보기) 점 반지름은 1.4로 쓴다. `aria-label`과 `title`이 완전히 같은 문구로 중복되면 `title`을 뺀다.
- 배너는 공용 `.banner` 컴포넌트를 쓰고 info/success/warning/danger 4톤을 지원한다. info 톤만 액션 텍스트에 틸을 허용(요약 라벨과 동일 선례)하고, success/warning/danger는 본문·액션 텍스트를 잉크로 고정한 채 색은 좌측 도트로만 표현한다.
- Toast는 `aria-live="polite"`를 쓰고 3초 기준으로 사라진다.

금지:

- glassmorphism
- gradient text
- hero metric
- 카드 안의 카드
- 컬러 리스트 스트라이프
- emoji icon
- 설명 없는 icon button
- 컴포넌트 내부 raw color/spacing 값
- 카드 내부 New 배지, 판단 라벨(답장 필요 등 분류 문구), 완료/나중에 버튼
- 상세 패널 액션의 상시 틸 fill(기본은 배경 없는 틸 아이콘, hover/focus에서만 틸 fill + 흰 아이콘 반전)

## 인터랙션

모든 조작 요소는 포커스, hover, reduced motion, 의미 있는 label, 44px 이상 hit area를 가진다. 포커스 링은 틸 `#14807A`, 2px, offset 2px, `:focus-visible` 전용으로 배경색과 무관하게 독립 레이어로 그린다. Gmail을 변경하는 액션은 결과 상태와 되돌리기 가능 여부를 보여준다.

읽기 전용 브리핑 상태는 무거운 신뢰 패널을 추가하지 않고, Gmail 변경 없음 문구로 충분히 설명한다.

금지:

- hover 8%, pressed 10-12% 값을 hover 레이어에 재사용(hover는 4%로 확정, pressed는 미구현 상태로 별도 결정 대상)
- 포커스 링을 `:focus`(마우스 클릭 포함)에 적용 — 반드시 `:focus-visible`
- Gmail 변경 액션에서 결과·Undo 가능 여부 생략
